# Пошаговый деплой сайта с регистрацией

Следуй шагам по порядку. Галочкой ✓ отмечай выполненное.

---

## Часть 1: Залить код в существующий репозиторий tasprep

### Шаг 1. Подключи папку к репозиторию tasprep (если ещё не подключена)
Открой PowerShell:

```powershell
cd C:\Users\Alpha\.cursor\ai-tracking
git remote -v
```

Если список пустой или там другой репозиторий — добавь tasprep (подставь свой GitHub username):

```powershell
git remote add origin https://github.com/ТВОЙ-USERNAME/tasprep.git
```

Если уже есть `origin` на tasprep — ничего не меняй.

### Шаг 2. Залей изменения на GitHub
```powershell
cd C:\Users\Alpha\.cursor\ai-tracking
git add .
git commit -m "Сайт с регистрацией и профилями"
git branch -M main
git push -u origin main
```

Если раньше уже пушил в этот репозиторий, достаточно:
```powershell
git add .
git commit -m "Сайт с регистрацией и профилями"
git push
```

✓ Код в репозитории tasprep на GitHub

## Часть 2: Vercel — выложить сайт

### Шаг 3. Зайди на Vercel
- Открой https://vercel.com
- Нажми **Sign Up** → **Continue with GitHub**
- Разреши доступ Vercel к GitHub

### Шаг 4. Импортируй проект
- Нажми **Add New…** → **Project**
- В списке найди репозиторий **tasprep** и нажми **Import**

### Шаг 5. Укажи корневую папку
- Напротив **Root Directory** нажми **Edit**
- Введи: **bilimbridge**
- Нажми **Continue**

### Шаг 6. Добавь переменные Firebase (обязательно)
- Раскрой блок **Environment Variables**
- Добавь по одной переменной (значения скопируй из своего файла `.env.local` в папке bilimbridge):

| Name | Value (твоё из .env.local) |
|------|----------------------------|
| `NEXT_PUBLIC_FIREBASE_API_KEY` | твой apiKey |
| `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` | tasprep-835b2.firebaseapp.com |
| `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | tasprep-835b2 |
| `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` | tasprep-835b2.firebasestorage.app |
| `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID` | 815145284920 |
| `NEXT_PUBLIC_FIREBASE_APP_ID` | 1:815145284920:web:... |

- У каждой переменной отметь: **Production**, **Preview**, **Development**
- Нажми **Add** после каждой

### Шаг 7. Запусти деплой
- Нажми **Deploy**
- Подожди 2–4 минуты

✓ Сайт собран и доступен по ссылке вида `название-проекта.vercel.app`

### Шаг 8. Проверь сайт
- Нажми на ссылку проекта (или открой её в браузере)
- Нажми **Войти** в шапке
- Должна открыться страница с вкладками **Вход** и **Регистрация**
- Попробуй зарегистрироваться по email и паролю или войти через Google

Если видишь «Firebase config is missing» — вернись к шагу 6 и проверь, что все 6 переменных добавлены в Vercel, затем в проекте нажми **Redeploy**.

## Часть 3: Свой домен (по желанию)

### Шаг 9. Добавить домен в Vercel
- В проекте на Vercel открой **Settings** → **Domains**
- Введи домен (например `tasprep.com`) → **Add**
- Vercel покажет, какие DNS-записи нужно создать

### Шаг 10. Настроить DNS у регистратора
- Зайди в панель, где куплен домен (Namecheap, Reg.ru и т.д.)
- Открой настройки DNS домена (Advanced DNS / Управление зоной)
- Добавь записи так, как написано в Vercel (обычно A-запись на `76.76.21.21` и CNAME для `www` на `cname.vercel-dns.com`)
- Сохрани и подожди 10–60 минут

✓ Сайт открывается по твоему домену
